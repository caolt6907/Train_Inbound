机器人智能培训是inbound项目我主要负责呼入模块、post提交语音文本模块、tts转换模块、读取配置模块和发送消息机制等几部分功能开发。
呼入模块：
	由于业务需要呼入模块不处理业务逻辑交后台处理。
	呼入接通进入通话线程，调用老赵的jar包提供的静态方法可以得到两个参数一个是主叫号码、一个是登入工号。通过主要号码读取redis(这个redis是在redis池里取得对象)上的
配置主要判断是不是在工作时间段如不是作为错误处理，如在工作时间通过post模块向后台提交开始通话信息，得到信息反馈(类型、wav文件、语音文本)把wav文件发送给fs接收用户的语音反馈。
进入通话的主循环根据用户的语音文本向后台提交后得到类型结果来执行结束通话、继续循环通话等操作如碰到用户直接挂机则走错误处理模块，每一次交互都要通过kafka要把各种消息反馈给后台
服务器以便于统计分析有一个变化就是把以前的用户手机号改成登入工号。这里有个需指出的是碰到播放语音文件要是找不到通过tts进行现时转换进行播放。
	
post模块:
	项目原来的是在呼入模块里实现语音文本的逻辑判断，但鉴于培训项目的特殊性把逻辑判断这块放在后台来解决。这里主要通过生成一个静态的方法把需要提交的用户文本进行utf-8编码
通过http协议的post方法把提交的参数转换成json数据提交到后台同时得到反馈包括:类型、播放的wav文件、语音文本等信息。

TTS转换模块：
	把后台反馈的语音文本通过第三方提供的方法进行文本到语音流合成得到wav文件进行播放。

主模块:
	创建redis的连接池以便各个线程调用，生成一个Socke连接服务和fs服务器之间是通过Socket连接的，就是我这里先启动一个SockectServer连接服务等待老赵那里的client接入(这个接入就是有通话进入)
,如有接入实际上我这里就new一个通话模块来处理具体这通电话具体业务直到这通电话结束。原来的SockectServer继续等待新的电话接入有接入继续new一个通话模块来处理。这个来实现和fs之间的多线程通话。
这个也要说明这个等待接入是异步的不影响下面的执行语句。作为一个纯Inbound的项目，我这里的主程序等待6秒的死循环。
